themispy_best_sample¶

Reads a Themis ensemble chain, likelihood file pair and determines the entry with the highest likelihood. Optionally finds the lowest chi-squared, the reduced chi-squared, and/or generates a fit_summaries.txt-style file from the best fit.

Example: $ themispy_best_sample -c Chain.dat -l Lklhd.dat

usage: themispy_best_sample [-h] -l LKLHD -c CHAIN [-q CHISQ] [-d DOF] [-f [F]] [-m [M]] [-v] [-n N] [-p PARAMETERS [PARAMETERS ...]]

Named Arguments¶

-l, --lklhd

<Required> Name of likelihoood file to read.

-c, --chain

<Required> Name of chain file to read.

-q, --chisq

Name of chi-squared file to read. Does not read this coherently with likelihood and chain.

-d, --dof

Number of degrees of freedom to assume for reduced chi-squared computation. Has no effect if -q,–chisq not set.

-f

Output a fit_summaries.txt file with the highest-likelihood point. If an argument is provided, it will be assumed to be the desired file name.

-m

Print initialization list appropriate for cutting and pasting into driver file. If a string is passed, it will assume the parameters are of the form M[0], M[1], etc., are to be set.

-v

Verbose mode is set, printing the chain location directly to standard out.

-n

Number of optimal samples to output

-p, --parameters

Selects a subset of parameters to output. Parameter ranges may be specified either via individual numbers (e.g., 0 1 4), comma separated lists (e.g., 0, 2, 5), ranges (e.g., 0-3), or combinations thereof. By default, all parameters are outputted.

themispy_chain_diagnostics¶

Computes various chain diagnostics for Themis ensemble MCMC chain data. Diagnostics generated include split-Rhat, mean rank, and variance rank plots. Typically split-Rhat larger than 1.03 implies biases may persist in the posteriors, though frequently this is overkill for parameter estimation if other diagnostics appear reasonable.

Expample: $ themispy_chain_diagnostics

usage: themispy_chain_diagnostics [-h] -c CHAIN [CHAIN ...] -l LKLHD [LKLHD ...] [-bf BURN_FRACTION] [--stride STRIDE] [-p PARAMETERS [PARAMETERS ...]] [-o OUT] [-s] [--hist] [--usetex]

Named Arguments¶

-c, --chain

<Required> Chain files to read in, can pass multiple. Must be equal to number of likelihood files to read in

-l, --lklhd

<Required> Lklhd files to read in, can pass multiple. Must be equal to number of chain files to read in

-bf, --burn-fraction, -w, --warmup

Fraction of chain to skip. Default: 0.5.

--stride, --nthin

How much to thin the chain by. The DEFAULT is 1, i.e. no thinning

-p, --parameters

Selects a subset of parameters to output. Parameter ranges may be specified either via individual numbers (e.g., 0 1 4), comma separated lists (e.g., 0, 2, 5), ranges (e.g., 0-3), or combinations thereof. By default, all parameters are outputted.

-o, --out

Output file name. Default: chain_diagnostics.out

-s, --save

Flag to turn on whether the save the processed likelihood and chain files, i.e. thinned and burn-in removed

--hist

Flag to create rank plots for the chains read in.

--usetex

Turns on using LaTex in the rendering. Note that this takes longer, but the result is MUCH better looking.

themispy_deo_diagnostics¶

Reads in tempering summary output from DEO samplers in Themis and generates desired diagnostic plots. Various presentation options and diagnostic selection may be set.

Example: $ themispy_deo_diagnostics -d tl

usage: themispy_deo_diagnostics [-h] [-ad ANNEALING_DATA_FILE] [-as ANNEALING_SUMMARY_FILE] [-o OUT] [--density] [--stream] [-d DIAGNOSTICS [DIAGNOSTICS ...]] [-mr MINIMUM_ROUND] [-Mr MAXIMUM_ROUND] [--usetex]

Named Arguments¶

-ad, --annealing-data-file

Annealing data file to be read in. Default: Annealing.dat

-as, --annealing-summary-file

Annealing summary file to be read in. Default: <annealing_data_file>+.summary

-o, --out

Output file name. Default: deo_diagnostics.png

--density

Sets the tempering level evolution plot to show the density of tempering levels via the colormap. Default: False.

--stream

Sets the colormap of the tempering level evolution plot to use a streamline colormap. Default: False.

-d, --diagnostics

Sets the list of diagnostic plots to generate. Takes a combination of the letters t, r, l, which may be specified in any order and in any number of option calls. The appropriate plots will be made in the order trl. Diagnostic options are: tempering level evolution (t), rejection rate evolution (r), and lambda vs. beta (l). Default: trl.

-mr, --minimum-round

Minimum round to show. Default: 0

-Mr, --maximum-round

Maximum round to show. Default: all rounds.

--usetex

Turns on using LaTex in the rendering. Note that this takes longer, but the result is MUCH better looking.

themispy_dterm_plots¶

Reads in a Themis chain file from a polarized image model fit and plots the D term posteriors. A single chain file must be provided.

Example: $ themispy_dterm_plots chain.dat -xs SP –usetex

usage: themispy_dterm_plots [-h] [-m MODEL] [-bf BURN_FRACTION] [-s SAMPLES] [-o OUT] [--no-grid] [-rs RESTRICT_STATION_LIST [RESTRICT_STATION_LIST ...]] [-xs EXCLUDE_STATION_LIST [EXCLUDE_STATION_LIST ...]] [--truths TRUTHS]
                            [--usetex] [-v]
                            chain_file

Positional Arguments¶

chain_file

<Required> Name of chain file to read.

Named Arguments¶

-m, --model

Model specification glob. May be a model tag file name or an ccm_mexico+ model specification glob. See the vis.image.construct_model_image_from_glob function documentation for format details. If contains a ‘.’ will assume that it is a tagvers-1.0 tag file. Default: model_image.tag

-bf, --burn-fraction

Fraction of chain to excise as burn in. Default: 0.5

-s, --samples

Number of samples to draw from the specified chain file. Default: 1000.

-o, --out

Output file name. Default: gain_plot.png

--no-grid

Turn off grid lines. Default: True.

-rs, --restrict-station-list

Name of stations to which to restrict the residual plot. The station codes shouldbe specified as used in the data file.

-xs, --exclude-station-list

Name of stations to exclude from residual plot. The station codes should be specifiedas used in the data file.

--truths

Set a truths file to read in and add to plot. The file is expected to contain D terms for each station on a single line formatted as ‘station’, ‘DR.real’, ‘DR.imag’, ‘DL.real’, ‘DL.imag’. Optionally, errors for each D term may be provided. Default: None.

--usetex

Turns on using LaTex in the rendering. Note that this takes longer, but the result is MUCH better looking.

-v, --verbose

Provides verbose output about model and plot construction.

themispy_gain_plots¶

Reads in a Themis gain file of any type and generates combinations of plots of the amplitude and phase corrections for each station. A single gain file must be provided.

Example: $ themispy_gain_plots gain_corrections.d -i -o gplot.png –usetex

usage: themispy_gain_plots [-h] [-i] [-dt] [-ns] [-gp GAIN_PLOTS [GAIN_PLOTS ...]] [-dts] [-o OUT] [--no-grid] [--usetex] gain_file

Positional Arguments¶

gain_file

<Required> Name of gain file to read.

Named Arguments¶

-i, --invert

Invert gains prior to plotting. This is as if the gains were applied to the data (heresy!).

-dt

Plots the gains gains the time difference relative to the first epoch instead of absolute time.

-ns, --no-station-labels

Remove station labels

-gp, --gain-plots

Sets the list of gain plots to generate. Takes a combination of the letters ‘a’ (amplitudes), ‘p’ (phases), which may be specified in any order and in any number of option calls. The appropriate plots will be made in the order ap.

-dts, --deprecated-timestamp

Flags the gain file as having a deprecated timestamp (seconds from 00UT, Jan 1, 2000 instead of J2000)

-o, --out

Output file name. Default: gain_plot.png

--no-grid

Turn off grid lines. Default: True.

--usetex

Turns on using LaTex in the rendering. Note that this takes longer, but the result is MUCH better looking.

themispy_generate_data¶

Reads in a uvfits file and produces the desired data files for Themis with various additions. These include the ability to scan average the data, addition of fractional systematic error components, addition of fixed systematic error components, flagging of stations, and removal of baselines below some cutoff.

Example: $ themispy_generate_data data.uvfits -d v -s -f 2.0

usage: themispy_generate_data [-h] -d DATA_TYPE [DATA_TYPE ...] [-s] [--avg-time AVG_TIME] [-f FRACTIONAL_SYSTEMATIC_ERROR] [-t THRESHOLD_SYSTEMATIC_ERROR] [-x EXCLUDE_STATION [EXCLUDE_STATION ...]] [-b MINIMUM_BASELINE]
                              [--no-debias] [-snr SNR_CUT] [--crosshand-SNR-cut CROSSHAND_SNR_CUT] [--visibility-SNR-cut VISIBILITY_SNR_CUT] [--amplitude-SNR-cut AMPLITUDE_SNR_CUT]
                              [--closure-phase-SNR-cut CLOSURE_PHASE_SNR_CUT] [-er5] [-xph] [-tarr TELESCOPE_ARRAY_FILE] [-eht2017] [-eht2018]
                              uvfits_files [uvfits_files ...]

Positional Arguments¶

uvfits_files

<Required> Name of uvfits files to be processed.

Named Arguments¶

-d, --data-type

Data types to generate. Accepted types include x,v,a,p for crosshands, complex visibilities, amplitudes, closure phases. May be specified in independent calls. Default: None.

-s, --scan-avg

Produce scan averaged data.

--avg-time

Timescale over which to average the data in seconds. Superseded by -s,–scan-avg. Default: None.

-f, --fractional-systematic-error

Fractional systematic error to add IN PERCENT. Default: None.

-t, --threshold-systematic-error

Constant systematic error threshold to add in mJy. Default: None.

-x, --exclude-station

List of stations to flag. Stations must be specified with the same station codes appearing in the uvfits file. E.g., the 2017 code list. Default: None.

-b, --minimum-baseline

Minimum baseline, below which data are excluded from ALL data generation. Default: None.

--no-debias

Do not debias visibility amplitude data. Default: debias amplitudes.

-snr, --SNR-cut

SNR cut to apply to all data products unless specific cuts are specified.

--crosshand-SNR-cut

SNR cut to apply to crosshand data.

--visibility-SNR-cut

SNR cut to apply to complex visibility data.

--amplitude-SNR-cut

SNR cut to apply to amplitude data.

--closure-phase-SNR-cut

SNR cut to apply to closure phase data.

-er5

Indicates that this is an ER5 data set that suffers from the polsolve bug and must be corrected. Default: False.

-xph, --exclude-partial-hands

Excludes partial hand data (e.g., JCMT) in the construction of crosshand visibility data sets. Default: include partials.

-tarr, --telescope-array-file

Reads a separate telescope array file.

-eht2017

Uses a default telescope array file for 2017.

-eht2018

Uses a default telescope array file for 2018.

themispy_image_family¶

Reads in a Themis MCMC run and generates images for a subsample. If a likelihood file is specified, the highest likelihood model will be plotted in the center. Optionally will generate individual images instead of one multi-panel plot.

Example: $ themispy_image_family -c Chain.dat -l Lklhd.dat –samples 9

usage: themispy_image_family [-h] -c CHAIN [-l LKLHD] [-s SAMPLES] [-m MODEL] [-r [REFERENCE [REFERENCE ...]]] [-bf BURN_FRACTION] [-N [N [N ...]]] [--limits LIMITS [LIMITS ...]] [-n NORM] [-t TRANSFER] [-nm] [-o OUT [OUT ...]]
                             [--uvfits UVFITS] [--smooth SMOOTH] [--usetex] [-v]

Named Arguments¶

-c, --chain

<Required> Name of chain file to read.

-l, --lklhd

Name of likelihood file to read. If given, will plot the highest likelihood sample in the center of the family.

-s, --samples

Number of samples to draw from the specified chain file. Default: 9.

-m, --model

Model specification glob. May be a model tag file name or an ccm_mexico+ model specification glob. See the vis.image.construct_model_image_from_glob function documentation for format details. If contains a ‘.’ will assume that it is a tagvers-1.0 tag file. Default: model_image.tag

-r, --reference

If the model type is a model_image_sum or model_polarized_image_sum, will fix the image origin of the specified component. If no components are specified, will choose the first. Default: no component will be fixed to the origin.

-bf, --burn-fraction

Fraction of chain to excise as burn in. Default: 0.5

-N

Number of pixels in image. May be a single integer (square image) or two integers (non-square image). If None uses the default in vis.image.plot_model_image function. Default: None.

--limits

Limits (in microarcseconds) of the plot region. May be a single float or a set of four floats. If None uses the default in vis.image.plot_model_image function. Default: None.

-n, --norm

Logarithmic normalization relative to to the median and maximum brightness. A value of 0.0 sets Imax to the median, 1.0 sets Imax to the maximum. Default 1.

-t, --transfer

Set the transfer function (‘linear’,’sqrt’,’log’). Default: ‘linear’

-nm, --no-multipanel

Plots individual panels instead of a single multipanel figure. Output file names will be appended with the integer number of the frame with index 0 corresponding to the best fit. Default: multiplanel plot.

-o, --out

Set output files. Multiple files and filetypes may be specified, including fits. However, if a filename ending in ‘.fits’ is passed, the path to a uvfits data file must be provided via –uvfits to set the appropriate header information. Default: fit_family.png

--uvfits

Specifies a uvfits file from which fits header information for the relevant sources may be extracted.

--smooth

If passed, will smooth the image with a symmetric Gaussian filter with FWHM given in microarcseconds.

--usetex

Turns on using LaTex in the rendering. Note that this takes longer, but the result is MUCH better looking.

-v, --verbose

Provides verbose output about model and plot construction.

themispy_image_fit¶

Reads in various Themis output files and generates a plot of the resulting image. A sample parameter value must be provided in one of three ways: (1) by specifying a fit_summaries.txt file, (2) by specifying an MCMC chain file with an index (optional), or (3) by specifying an MCMC chain file and corresponding likelihood file. This may be saved in ways, including standard image formats (PNG,JPG,PDF) and FITS.

Example: $ themispy_image_fit -c Chain.dat -l Lklhd.dat

usage: themispy_image_fit [-h] [-c CHAIN] [-l LKLHD] [-f FSFILE] [-i INDEX] [-m MODEL] [-r [REFERENCE [REFERENCE ...]]] [-N [N [N ...]]] [--limits LIMITS [LIMITS ...]] [-n NORM] [-t TRANSFER] [--movie [MOVIE [MOVIE ...]]]
                          [--timestamp-file] [-o OUT [OUT ...]] [--uvfits UVFITS] [-s SMOOTH] [--usetex] [-v]

Named Arguments¶

-c, --chain

Name of chain file to read. If given, either an index must be specified with -i,–index or likelihood file must be specified with -l,-lklhd.

-l, --lklhd

Name of likelihood file to read. If given, will plot the highest likelihood sample. Requires chain file to be provided with -c,–chain.

-f, --fsfile

Name of fit summaries file to read. If given, will plot records listed within. If an index is given with -i,–index will plot the corresponding line; otherwise plots the first line. Default: fit_summaries.txt

-i, --index

Name of fit summaries file to read. If given, will plot records listed within. If an index is given with -i,–index will plot the corresponding line; otherwise plots the first line.

-m, --model

Model specification glob. May be a model tag file name or an ccm_mexico+ model specification glob. See the vis.image.construct_model_image_from_glob function documentation for format details. If contains a ‘.’ will assume that it is a tagvers-1.0 tag file. Default: model_image.tag

-r, --reference

If the model type is a model_image_sum or model_polarized_image_sum, will fix the image origin of the specified component. If no components are specified, will choose the first. Default: no component will be fixed to the origin.

-N

Number of pixels in image. May be a single integer (square image) or two integers (non-square image). If None uses the default in vis.image.plot_model_image function. Default: None.

--limits

Limits (in microarcseconds) of the plot region. May be a single float or a set of four floats. If None uses the default in vis.image.plot_model_image function. Default: None.

-n, --norm

Logarithmic normalization relative to to the median and maximum brightness. A value of 0.0 sets Imax to the median, 1.0 sets Imax to the maximum. Default: 1.0.

-t, --transfer

Set the transfer function (‘linear’,’sqrt’,’log’). Default: ‘linear’

--movie

Will treat the model as time variable. Arguments are optional, and may be contain a [start]:[step]:[stop] sequence, containing any combination of start, step, stop times in hours (colons required). Missing values will be set to defaults: if –uvfits is set, the start/stop times will be set to the first and last timesteps therein, otherwise the default start/stop times are 0h and 24h. In both cases, the default time step is 1h.

--timestamp-file

If saving multiple frames, will append filename with timestamp instead of index. Default: frames will be indexed.

-o, --out

Set output files. Multiple files and filetypes may be specified, including fits. However, if a filename ending in ‘.fits’ is passed, the path to a uvfits data file must be provided via –uvfits to set the appropriate header information. Default: fit.png

--uvfits

Specifies a uvfits file from which fits header information for the relevant sources may be extracted.

-s, --smooth

If passed, will smooth the image with a symmetric Gaussian filter with FWHM given in microarcseconds.

--usetex

Turns on using LaTex in the rendering. Note that this takes longer, but the result is MUCH better looking.

-v, --verbose

Provides verbose output about model and plot construction.

themispy_polimage_fit¶

A sample parameter value must be provided in one of three ways: (1) by specifying a fit_summaries.txt file, (2) by specifying an MCMC chain file with an index (optional), or (3) by specifying an MCMC chain file and corresponding likelihood file. This may be saved in ways, including standard image formats (PNG,JPG,PDF) and FITS.

Example: $ themispy_polimage_fit -c Chain.dat -l Lklhd.dat

usage: themispy_polimage_fit [-h] [-p PLOT_TYPE] [-c CHAIN] [-l LKLHD] [-f FSFILE] [-i INDEX] [-m MODEL] [-r [REFERENCE [REFERENCE ...]]] [-N [N [N ...]]] [-tN [TN [TN ...]]] [--limits LIMITS [LIMITS ...]] [-t TRANSFER]
                             [-tif TICK_INTENSITY_FLOOR] [--movie [MOVIE [MOVIE ...]]] [--timestamp-file] [-o OUT [OUT ...]] [--uvfits UVFITS] [-s SMOOTH] [-ms MSCALE] [--usetex] [-v]

Named Arguments¶

-p, --plot-type

Sets type of polarized image plot to generate. Options are ‘lf’, ‘ef’, ‘lL’. In detail: ‘lf’ produces linear polarization ticks, color-coded by linear polarization fraction, on top of total flux; ‘ef’ produces elliptical polarization ticks, color-coded by total polarization fraction, on top of total flux; ‘lL’ produces linear polarization ticks on top of linear polarized flux, with total flux contours superimposed. Default: ‘lf’.

-c, --chain

Name of chain file to read. If given, either an index must be specified with -i,–index or likelihood file must be specified with -l,-lklhd.

-l, --lklhd

Name of likelihood file to read. If given, will plot the highest likelihood sample. Requires chain file to be provided with -c,–chain.

-f, --fsfile

Name of fit summaries file to read. If given, will plot records listed within. If an index is given with -i,–index will plot the corresponding line; otherwise plots the first line. Default: fit_summaries.txt

-i, --index

Name of fit summaries file to read. If given, will plot records listed within. If an index is given with -i,–index will plot the corresponding line; otherwise plots the first line.

-m, --model

Model specification glob. May be a model tag file name or an ccm_mexico+ model specification glob. See the vis.image.construct_model_image_from_glob function documentation for format details. If contains a ‘.’ will assume that it is a tagvers-1.0 tag file. Default: model_image.tag

-r, --reference

If the model type is a model_image_sum or model_polarized_image_sum, will fix the image origin of the specified component. If no components are specified, will choose the first. Default: no component will be fixed to the origin.

-N

Number of pixels in image. May be a single integer (square image) or two integers (non-square image). If None uses the default in vis.image.plot_model_image function. Default: None.

-tN

Number of polarization ticks on image. May be a single integer (square image) or two integers (non-square image). If None uses the default in the vis.image.plot_polarized_image or vis.image.plot_linear_polarization_map functions. Default: None.

--limits

Limits (in microarcseconds) of the plot region. May be a single float or a set of four floats. If None uses the default in vis.image.plot_model_image function. Default: None.

-t, --transfer

Set the transfer function (‘linear’,’sqrt’,’log’). Default: ‘linear’

-tif, --tick-intensity-floor

Fraction of the maximum intensity above which to plot polarization ticks.Default 0.1.

--movie

Will treat the model as time variable. Arguments are optional, and may be contain a [start]:[step]:[stop] sequence, containing any combination of start, step, stop times in hours (colons required). Missing values will be set to defaults: if –uvfits is set, the start/stop times will be set to the first and last timesteps therein, otherwise the default start/stop times are 0h and 24h. In both cases, the default time step is 1h.

--timestamp-file

If saving multiple frames, will append filename with timestamp instead of index. Default: frames will be indexed.

-o, --out

Set output files. Multiple files and filetypes may be specified, including fits. However, if a filename ending in ‘.fits’ is passed, the path to a uvfits data file must be provided via –uvfits to set the appropriate header information. Default: fit.png

--uvfits

Specifies a uvfits file from which fits header information for the relevant sources may be extracted.

-s, --smooth

If passed, will smooth the image with a symmetric Gaussian filter with FWHM given in microarcseconds.

-ms, --mscale

Sets the polarization fraction normalization. Default: 0.5.

--usetex

Turns on using LaTex in the rendering. Note that this takes longer, but the result is MUCH better looking.

-v, --verbose

Provides verbose output about model and plot construction.

themispy_resample¶

Reads a Themis ensemble chain file, and optionally a likelihood file and samples it the desired number of times after a specified burn-in period. Optionally generates a resampled likelihood file.

Example: $ themispy_resample -c Chain.dat -l Lklhd.dat -s 1000 -bf 0.5

usage: themispy_resample [-h] -c CHAIN [CHAIN ...] [-l LKLHD [LKLHD ...]] [-s SAMPLES] [-bf BURN_FRACTION] [--postfix POSTFIX] [-p PARAMETERS [PARAMETERS ...]]

Named Arguments¶

-c, --chain

<Required> Name of chain file to read. If multiple files are listed, they will independently be sampled.

-l, --lklhd

Name of likelihoood file to read. If multiple files are listed, they will independently be sampled in order with the chain files. That is, the first likelihood file will be coherently sampled with the first chain file, the second likelihood file will be coherently sampled with the second chain file, etc.

-s, --samples

Number of samples to draw from the chain.

-bf, --burn-fraction

Fraction of chain to skip.

--postfix

Postfix to add to chain [and likelihood] file names produced after sampling.

-p, --parameters

Selects a subset of parameters to output. Parameter ranges may be specified either via individual numbers (e.g., 0 1 4), comma separated lists (e.g., 0, 2, 5), ranges (e.g., 0-3), or combinations thereof. By default, all parameters are sampled.

themispy_residual_plots¶

Reads in a Themis residual file of any type and generates plots of the residuals. A gain file may be provided and used to “calibrate” the data, though this requires a Themis-style data file to be provided. For polarized models, D terms may be applied to the data. A single residual file must be provided.

Example: $ themispy_residual_plots residuals.d –plot-type uvamp|amplitude -o residuals.png –usetex

usage: themispy_residual_plots [-h] [-g GAINFILE] [-d DATAFILE] [-rs RESTRICT_STATION_LIST [RESTRICT_STATION_LIST ...]] [-xs EXCLUDE_STATION_LIST [EXCLUDE_STATION_LIST ...]] [-pt PLOT_TYPE]
                               [--crosshand CROSSHAND [CROSSHAND ...]] [-o OUT] [--no-grid] [--no-res] [--no-resdist] [--usetex] [-D] [-c CHAINFILE] [-l LKLHDFILE] [-f FSFILE] [-i INDEX] [-m MODEL] [-v]
                               residual_file

Positional Arguments¶

residual_file

<Required> Name of residual file to read.

Named Arguments¶

-g, --gainfile

Name of Themis gain file to read. This will be applied to all of the residuals.

-d, --datafile

Name of Themis data file to read. This will be applied to all of the residuals.

-rs, --restrict-station-list

Name of stations to which to restrict the residual plot. The station codes shouldbe specified as used in the data file.

-xs, --exclude-station-list

Name of stations to exclude from residual plot. The station codes should be specifiedas used in the data file.

-pt, --plot-type

Plot type to generate (see ThemisPy documentation for residual-type-specific options.This will be applied to all residual plots and thus may be fragile if too specific. Ifa pipe is included in the specification, e.g., ‘time|amplitude’, the argument shouldbe past with quotes.

--crosshand

If residuals are being plotted for polarized image data, define which crosshands to plot.

-o, --out

Output file name. Default: gain_plot.png

--no-grid

Turn off grid lines. Default: True.

--no-res

Turn off residual plot. Default: residuals are plotted.

--no-resdist

Turn off residual distribution plot. Default: residual distribution is plotted. Onlyrelevant if residuals are plotted.

--usetex

Turns on using LaTex in the rendering. Note that this takes longer, but the result is MUCH better looking.

-D, --Dterms

If this is a polarized model, apply fitted D terms.

-c, --chainfile

Name of Themis chain file to read. Necessary if applying D terms.

-l, --lklhdfile

Name of fit summaries file to read. If given, will plot records listed within. If an index is given with -i,–index will plot the corresponding line; otherwise plots the first line. Default: fit_summaries.txt

themispy_selfcal_uvfits¶

Reads in a Themis gain file of any type and the matching uvfits data file and writes the self-calibrated data file.

Example: $ themispy_selfcal_uvfits -g gain_corrections.d -d data.uvfits

usage: themispy_selfcal_uvfits [-h] -g G -d UVFITS [-s] [--avg-time AVG_TIME] [-r] [-f FRACTIONAL_SYSTEMATIC_ERROR] [-t THRESHOLD_SYSTEMATIC_ERROR] [-b MINIMUM_BASELINE] [-x EXCLUDE_STATION [EXCLUDE_STATION ...]] [-o OUT]
                               [-v VERBOSE]

Named Arguments¶

-g

<Required> Name of gain file to read.

-d, --uvfits

<Required> Name of uvfits data file.

-s, --scan-avg

Produce scan averaged data.

--avg-time

Timescale over which to average the data in seconds. Superseded by -s,–scan-avg. Default: None.

-r, --relative-timestamps

Will ignore the absolute time offset in the gain file. Requires that the number of data points in the uvfits file match the number of gain reconstruction epochs.

-f, --fractional-systematic-error

Fractional systematic error to add IN PERCENT. Default: None.

-t, --threshold-systematic-error

Constant systematic error threshold to add in mJy. Default: None.

-b, --minimum-baseline

Minimum baseline, below which data are excluded from ALL data generation. Default: None.

-x, --exclude-station

List of stations to flag. Stations must be specified with the same station codes appearing in the uvfits file. E.g., the 2017 code list. Default: None.

-o, --out

Output file name. Default: gain_plot.png

-v, --verbose

Sets verbosity level in :fun:`write_uvfits`. Level 1 outputs values, level 4 generates multiple plots.

themispy_trace_plots¶

Reads a Themis ensemble chain file, and optionally a likelihood file, and generates trace plots after a specified burn-in period. If likelihood files are provided, the output will be annotated trace plots.

Example: $ themispy_trace_plots -c Chain.dat.r* -l Lklhd.dat.r* -s 10 –stream -p 0-4

usage: themispy_trace_plots [-h] -c CHAIN [CHAIN ...] [-l LKLHD [LKLHD ...]] [-s STRIDE] [-bf BURN_FRACTION] [-p PARAMETERS [PARAMETERS ...]] [-w WALKERS] [-o OUT] [-lo LIKELIHOOD_OUT] [--no-grids] [-m] [-G] [-1] [-P]
                            [--usetex]

Named Arguments¶

-c, --chain

<Required> Name of chain file(s) to read. If multiple files are listed, the burn-in will be independently be applied.

-l, --lklhd

Name of likelihoood file to read. If present, a matching likelihood file must be provided for every chain file.

-s, --stride

Stride over which to sample the chain.

-bf, --burn-fraction

Fraction of chain to skip.

-p, --parameters

Selects a subset of parameters to output. Parameter ranges may be specified either via individual numbers (e.g., 0 1 4), comma separated lists (e.g., 0, 2, 5), ranges (e.g., 0-3), or combinations thereof. By default, all parameters are sampled.

-w, --walkers

Number of walkers. Only necessary if likelihood files are not provided.

-o, --out

Output file name. Default: parameter_trace_plot.png

-lo, --likelihood-out

Output file name. Default: likelihood_trace_plot.png

--no-grids

Turn off grid lines. Default: True.

-m, --means

Turn on the plotting of means. Default: False.

-G, --Global

Sets the likelihood colormap to use the global limits on the likelihood. Only meaningful if likelihood files are passed. Default: False.

-1, --one-column

Organizes trace plots into a single column. Default: False.

-P, --parameters-only

Disable likelihood plot on the parameter trace plot. When likelihoods are passed, a likelihood trace is added to the traces of the parameters. Default: likelihood trace is added.

--usetex

Turns on using LaTex in the rendering. Note that this takes MUCH longer, but the result is MUCH better looking.

themispy_triangle_plot¶

Reads a Themis chain file and produces triangle plots. The parameters to show may be specified. Up to two chain files may be specified, which will be plotted on lower and upper triangles, respectively. If a model glob is given, the appropriate labels will be set. If a model is set, setting -deg will attempt to specify angles in the appropriate degree format (e.g., deg, arcmin, arcsec, mas, etc.).
